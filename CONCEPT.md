# Концепция приложения «Сходка» (рабочее название)

## 1. Идея в одном предложении

Приложение, в котором любой человек может за минуту создать заявку «хочу сделать X тогда-то там-то, ищу компанию» — про абсолютно любую активность (гидроциклы, теннис, концерт, прогулка, настолки, поход в музей), а другие пользователи рядом видят это в ленте и могут откликнуться.

## 2. Проблема, которую решаем

Сейчас компанию ищут либо:
- в узких тематических сервисах (только спорт, только путешествия, только знакомства);
- в случайных группах ВК/Telegram, где объявление быстро тонет в потоке;
- через личные связи, которых может не хватать на конкретную активность в конкретный день.

Нет универсального места, куда можно прийти с любой идеей «хочу это сделать сегодня/в выходные» и быстро найти 2-5 человек со схожим желанием.

## 3. Целевая аудитория

- Люди 20-40 лет в крупных и средних городах, у которых гибкий график и желание проводить время активно/разнообразно.
- Переехавшие в новый город / фрилансеры / удалёнщики — у них меньше устоявшегося круга общения.
- Люди, у которых есть идея на выходные, но их компания «не в настроении» или занята.

## 4. Как это работает (пользовательский путь)

1. **Создание события.** Пользователь заполняет короткую форму: что делаем, когда, где, сколько человек нужно, краткое описание.
2. **Публикация в ленту.** Событие появляется в общей ленте у пользователей рядом (по геолокации) и подходящих по интересам.
3. **Отклик.** Заинтересованные отправляют заявку на участие («хочу с вами»).
4. **Подбор группы.** Организатор подтверждает участников (либо включён авто-приём, если хватает места).
5. **Чат группы.** Автоматически создаётся групповой чат для согласования деталей (где встречаемся, кто что берёт).
6. **После события.** Взаимные короткие отзывы/рейтинг — для повышения доверия в системе.

## 5. Структура карточки события

| Поле | Описание |
|---|---|
| Активность | Свободный текст («Гидроциклы», «Сходить на концерт») + опциональные теги/категории для поиска |
| Дата и время | Конкретное время или диапазон («после обеда») |
| Локация | Точка на карте + текстовое описание (адрес, прокат, место встречи) |
| Кол-во участников | Сколько уже есть / сколько нужно ещё (мин-макс) |
| Описание | Свободный текст: формат, что взять с собой, уровень подготовки, бюджет |
| Стоимость | Если есть — общая цена и как делится (например, прокат гидроцикла на компанию) |
| Фото/обложка | Необязательно, для атмосферы |
| Предпочтения по составу | Опционально: компания/пол/возраст — но без обязательной фильтрации, чтобы не превращать в формат знакомств |

## 6. Лента и поиск

- **Карта + список** — переключение между видами.
- **Быстрые фильтры:** «сегодня», «завтра», «на выходных», расстояние от меня, категория активности.
- **Поиск по тексту/тегам** — для тех, кто ищет конкретное («теннис», «концерт», «настолки»).
- **Уведомления по подписке** — можно подписаться на категорию или район и получать пуш, когда там появляется новое событие.

## 7. Доверие и безопасность

Это самый чувствительный момент — встречи с незнакомцами требуют доверия:

- Верификация через номер телефона (обязательно) и привязку соцсети (опционально, но повышает доверие — видно общих друзей, если интеграция с VK/др.).
- Рейтинг и короткие отзывы после события — двусторонние (организатор оценивает участников, участники — организатора и мероприятие).
- Жалобы и блокировка пользователей.
- Формат «группа», а не «один на один» — сам по себе снижает риски по сравнению с классическими сервисами знакомств.
- Видимость истории: сколько событий человек создал/посетил, давно ли в приложении — помогает отличать «живых» пользователей от случайных профилей.

## 8. Механика заявок и подбора

- Организатор видит список откликнувшихся (краткий профиль + рейтинг) и подтверждает нужное количество.
- Можно включить «авто-приём первых N» — для тех, кто не хочет тратить время на отбор.
- Если место освобождается (кто-то отказался), следующий из списка ожидания получает уведомление.

## 9. Чат и координация

- При формировании группы автоматически создаётся чат только для подтверждённых участников.
- В чате — закреплённое сообщение с деталями события (время, место, что взять).
- После завершения события чат автоматически архивируется (но доступен для истории).

## 10. Чем отличается от существующих решений

| Существующие сервисы | «Сходка» |
|---|---|
| Узкая тематика (только спорт, только путешествия, только знакомства) | Любая активность — от гидроциклов до похода в кино |
| Формат «анкета + переписка» | Формат «событие + заявки», как в афишах, но создавать может любой |
| Долгосрочное планирование (поездки на недели) | В первую очередь про «здесь и сейчас» / ближайшие дни |
| Нужно искать тематическое сообщество | Один общий поток рядом с тобой |

## 11. Монетизация

- **Freemium:** базовый функционал бесплатно; платно — поднятие события в ленте, расширенные фильтры, больше одновременных активных событий.
- **Партнёрства с локальным бизнесом:** прокат гидроциклов, квесты, кафе, скалодромы могут публиковать «готовые слоты» с возможностью онлайн-бронирования через приложение (комиссия с бронирования).
- **Премиум-подписка:** без рекламы, приоритетное появление в ленте, расширенная статистика по своим событиям.
- **Локальная реклама:** в ленте рядом с событием — релевантные предложения («рядом с местом встречи есть кафе со скидкой для вашей группы»).

## 12. MVP — минимальный функционал первой версии

1. Регистрация через телефон.
2. Создание события (упрощённая форма: что/когда/где/сколько человек/описание).
3. Лента событий рядом (список + простая карта).
4. Заявка на участие + подтверждение организатором.
5. Групповой чат.
6. Базовый профиль (имя, фото, короткое «о себе», рейтинг после первых событий).

Всё остальное (категории-подписки, бронирование у партнёров, продвинутые фильтры) — во второй итерации, когда понятно, что базовый цикл «создал → откликнулись → встретились» работает.

## 13. Технический стек (вариант для MVP)

- **Backend:** Node.js/NestJS или Python/FastAPI — быстрый старт, много готовых библиотек.
- **База данных:** PostgreSQL + PostGIS (геопоиск событий рядом).
- **Мобильное приложение:** React Native или Flutter — единая кодовая база для iOS и Android.
- **Чаты:** готовое решение (например, Firebase / собственный на WebSocket) — не изобретать велосипед на старте.
- **Push-уведомления:** Firebase Cloud Messaging.
- **Хостинг:** любой облачный провайдер с возможностью масштабирования (старт — минимальная конфигурация).

## 14. Основные риски

- **Проблема холодного старта:** мало пользователей → мало событий → новым пользователям нечего смотреть → они уходят. Решение — запуск в одном городе/районе, точечный маркетинг через существующие сообщества (студенческие, спортивные, экспат-чаты), возможно — «затравочные» события от самой команды на старте.
- **Безопасность:** нужна чёткая политика модерации, быстрый разбор жалоб, понятные правила.
- **Спам и фейковые события:** лимиты на создание, верификация, репутационная система.
- **Сезонность активностей:** часть идей завязана на погоду/сезон — важно показывать актуальные категории (зимой — каток/лыжи, летом — вода/пикники).

## 15. Возможные названия

- «Сходка»
- «Го» / «Гоу»
- «Компания»
- «Вместе»
- «Слот» (в смысле «свободный слот в планах»)
- «Рядом» (с акцентом на геолокацию)

## 16. Дальнейшее развитие (после MVP)

- Рекомендации событий на основе истории интересов (простая ML-модель).
- Календарная синхронизация — напоминания о предстоящих событиях.
- Командные/групповые профили — для постоянных компаний, которые регулярно зовут новых людей.
- Интеграция оплаты — сплит счёта внутри группы (прокат, аренда, билеты).
- Веб-версия для удобного создания событий с компьютера.
